# Version Tracking in Conformance Reports

This C2PA Conformance Tool automatically includes version information in every report to help track which version of the tool was used to generate each report.

## What's Included

Every conformance report includes a `_conformanceToolVersion` field with the following information:

```json
{
  "_conformanceToolVersion": {
    "commit": "7bf7a937a6fc3b751fb693c5a64e425f8c55900d",
    "shortCommit": "7bf7a93",
    "date": "2026-02-11 08:54:29 -0500",
    "branch": "main",
    "generatedAt": "2026-02-11T16:49:52.969Z"
  }
}
```

### Fields

- **commit**: Full git commit SHA (40 characters) - uniquely identifies the exact code version
- **shortCommit**: Abbreviated commit SHA (7 characters) - convenient short reference
- **date**: Commit date in ISO format with timezone - when the code was committed
- **branch**: Git branch name - typically "main" for production builds
- **generatedAt**: Timestamp when the version file was generated - when the build occurred

## Where to Find It

### In the UI
The version information is **not displayed** in the visual report UI to keep it clean and focused on C2PA data.

### In the JSON View
1. Upload a C2PA file
2. Scroll down to the "Raw JSON Report" section
3. Look for the `_conformanceToolVersion` field at the root level

### In Downloaded Reports
When you click the "Download JSON" button, the downloaded file will include the `_conformanceToolVersion` field with all version information.

## How It Works

### Build Time
1. When you run `npm run dev` or `npm run build`, the `scripts/generate-version.js` script runs automatically
2. The script executes git commands to capture current repository information
3. It generates `src/lib/version.ts` with the version data
4. This file is imported by `src/lib/c2pa.ts` and included in every report

### Runtime
When processing a C2PA file, the `processFile()` function automatically adds the version information to the manifest store before returning it.

## Benefits

1. **Reproducibility**: Know exactly which code version produced each report
2. **Debugging**: Quickly identify if issues are version-specific
3. **Auditing**: Track when and how reports were generated
4. **Comparison**: Compare reports from different tool versions

## Development Notes

- The `src/lib/version.ts` file is auto-generated and excluded from git (in `.gitignore`)
- If git is not available, the script will gracefully use "unknown" for all fields
- The version info uses a `_` prefix to indicate it's metadata, not C2PA data
- The structure is designed to be easily parsed by automated tools

## Example Usage

If you're processing reports programmatically:

```javascript
// Load a downloaded report
const report = JSON.parse(reportJsonString)

// Check the version
if (report._conformanceToolVersion) {
  console.log(`Report generated by commit ${report._conformanceToolVersion.shortCommit}`)
  console.log(`on ${report._conformanceToolVersion.date}`)
}

// Filter reports by version
const reportsFromMain = allReports.filter(r =>
  r._conformanceToolVersion?.branch === 'main'
)
```

## Troubleshooting

**Q: I don't see version info in my report**
A: Make sure you ran `npm run build` or `npm run dev` which generates the version file automatically.

**Q: Version shows "unknown"**
A: This happens when git is not available or the directory is not a git repository. The tool will still work, but version tracking won't be available.

**Q: Can I disable version tracking?**
A: The version info is harmless metadata. If you need to remove it, you can modify `src/lib/c2pa.ts` to not include the `_conformanceToolVersion` field.
